📐 SWE-bench Verified 方法论 评估框架说明

📄 论文来源

  • Jimenez 等人发表的论文 "Can Language Models Resolve Real-World GitHub Issues?"
  • 数据集：SWE-bench Verified 子集，由 OpenAI 于 2024 年 8 月发布
  • 每个任务经人类专家审验，500 个高质量测试用例
要素说明
Agent 框架mini-swe-agent，最小化 bash-tool-only agent harness
可用工具单一工具：bash。模型必须使用标准命令行工具（grep、find、sed 等）导航代码库、搜索文件并应用编辑
数据集SWE-bench Verified 子集，500 个经人类验证的高质量测试用例
配置使用提供商默认配置，max token limit 设为最高可能值
运行环境隔离的云沙箱
延迟计算从模型在每个任务中采取的第一步开始计算
评判标准对生成的 patch 运行单元测试来确定成功与否
公平性原则所有模型使用相同工具，确保公平对比

⚠️ 注意事项

  • SWE-bench 同时评估 agentic harness 和底层基础模型
  • 不同实验室报告结果时采用的方法论不同
  • 计算需求使得复现结果资源密集

ℹ️ 关于 Arena AI 盲测机制与 Elo 评分方法

⚔️ 什么是 Arena AI 盲测？

  • Arena AI（原 LMSYS Chatbot Arena）采用盲测对比机制：用户输入提示后，两个匿名模型同时生成回答
  • 用户根据回答质量投票选择更优的一方，投票后才会揭晓模型身份
  • 这种机制有效消除了品牌偏见，反映模型的真实使用体验
  • 截至目前已累计超过 600 万票的人类偏好数据

📊 Elo 评分方法

  • 采用国际象棋等竞技比赛中广泛使用的 Elo 评分系统
  • 每个模型初始分数为 1000，每次对局后根据结果和对手评分进行调整
  • 击败高分对手获得更多分数，输给低分对手损失更多分数
  • Elo 评分后的 ±数值表示 95% 置信区间，数值越小表示评分越稳定

关于 TERMS-Bench

TERMS-Bench 是一个用于大语言模型（LLM）谈判智能体的诊断性基准测试，来源于论文 arXiv:2605.13909。

🔬 核心设计原则

  • 模拟器驱动：对手是固定的随机策略（而非另一个 LLM），因此每个回合都可从种子复现，代理间比较干净公平
  • 无人工/LLM 裁判：所有指标均从记录的行动中程序化计算
  • 无综合评分：四个正交诊断轴独立报告，不合成复合分数
  • 双边价格谈判：每个智能体在三种机制和六个行为家族中与固定的、历史反应型对手进行多轮交替报价

🎮 游戏机制

  • 单一双边价格谈判，针对一件物品
  • 每方持有私有类型 t = (r, κ, η) — 保留价格、紧迫性、立场
  • 最多 K = 10 轮的交替报价
  • 每轮代理接收对手的价格 + 情绪 + 立场提示
  • 代理返回 Offer(price, message)、Accept 或 Reject

📐 四个正交诊断轴

  • SE⁺ 可行剩余效率 — 捕获的可用谈判剩余比例（越高越好）
  • AGR⁺ 可行协议率 — 可行回合中达成协议的比例（越高越好）
  • CSE⁺ 条件可行交易质量 — 以达成协议为条件的 SE⁺（越高越好）
  • FAGR⁻ 无交易假协议率 — 不可行回合中仍然同意的比例（越低越好）
  • BEtype 信念误差 — 对手保留价值、紧迫性和立场的信念聚合误差（越低越好）
  • CritVol% 严重违规率 — 价格边界/个体理性/无效行动违规回合比例（越低越好）

🎭 六个对手行为家族

雷达图/多边形图的六个辐条：Candid（坦率）、Taciturn（沉默）、Expressive（表现力）、Strategic（策略性）、Stochastic（随机）、Adversarial（对抗性）
